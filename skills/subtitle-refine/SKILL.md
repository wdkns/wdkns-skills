# 字幕精修

## 先读原始要求

先读取 [references/original-instructions.md](references/original-instructions.md)。

该文件保存用户给出的原始要求与补充说明。执行时只调整格式，不改动其中的内容表达。具体到当前任务时，把其中的示例文件名替换成用户本次实际提供的文件名即可。

## 工作原则

只做字幕级清洗和纠错，不做书面化改写、总结或扩写，即如无必要（即错误或者语气词），不对一些口语化的字词做删减。

保守优先。除非能明确判断为 ASR 识别错误、无意义语气词、明显重复口误、缺少必要停顿空格，或因超过 14 字必须拆分，否则不得改写原句语序、句法结构、搭配关系或替换成更顺的说法。

不得仅因“更通顺”“更简洁”“更像书面语”而改写原句。像“给我支持”改成“支持我”、“跟我说”改成“对我说”、“就是也是”改成“也是”等，若不属于明确识别错误或规则要求，一律不改。

优先保留原始口语表达。允许字幕不够书面、不够漂亮；只修错误，不做润色。若一句话在当前形态下已经满足规则，则必须保持原样。

删除规则从严。纯语气词独立条目可直接删除；除此之外，只有在能明确判断为断裂残句、重复残片或纯噪声误识别时才允许整条删除，并且必须把原始编号加入 allowlist，在交付中逐条列出。

不要覆盖原始字幕文件。始终输出新的清洗版文件，方便后续回听、校准和比对。

若文件很长，且用户明确要求并行或允许并行子代理，则按字幕编号或时间范围切块处理；各块独立清洗，最后统一合并和校验。

分块处理中，块内清洗结果默认保留原始 block_id；只有单条拆分时才使用 `123a`、`123b` 这类临时编号。不要在块内把字幕整体重编号，也不要为了美观调整编号顺序。

若只是文本清洗、纠错、删语气词、删重复字眼、补空格停顿，且未改变语义边界，则默认保持该条原始起止时间不变。

若必须拆分字幕，则拆分后的各子条目必须完全落在原条目的时间范围内，不得侵占后续条目时间。

不得跨条借词、补词或改写来“修顺”当前条目。若当前条目听不清或疑似 ASR 错乱，但无法仅依据本条明确修正，则优先保留原句并标记人工复听，不要从前后字幕主观拼接新文本。

## 推荐流程

1. 读取原始 `srt`，确认总条数、是否存在超长条目、是否需要按块处理。
2. 按 [references/original-instructions.md](references/original-instructions.md) 逐条执行清洗、纠错、必要拆分和时间轴控制。
3. 若采用分块处理，块内保留原编号；若单条拆分，可使用 `123a`、`123b` 这类临时编号。
4. 对非 filler 删除条目先整理 allowlist，再运行校验脚本。
5. 若已经产出多个 `clean_chunk_*.srt`，再运行 `scripts/merge_clean_srt.py` 合并为最终连续编号版本。
6. 运行 `scripts/check_clean_srt.py` 做规则检查和时间轴复核。
7. 对告警做人工复核，重点看语气词残留、重复口误残留、可能缺少停顿空格、过短闪现、文本串位和允许删除条目列表。

## 脚本资源

### `scripts/check_clean_srt.py`

用于检查清洗结果是否满足长度、标点、时间轴、删除条目、局部拆分边界等要求。

该脚本必须兼容两类输入：

- 已经拆分过的中间结果：允许保留原编号或使用 `123a`、`123b` 这类临时编号。
- 没有拆分过的清洗结果：即使字幕文件完全未拆分，也必须正常校验，不能假设一定存在拆分条目。

常用方式：

```bash
python3 scripts/check_clean_srt.py raw.srt clean.srt
python3 scripts/check_clean_srt.py raw.srt clean.srt --allowed-deletions '284,415,450'
python3 scripts/check_clean_srt.py raw.srt clean.srt --allowed-deletions '284,415,450' --fail-on-warnings
```

### `scripts/merge_clean_srt.py`

仅在分块处理后需要把多个 `clean_chunk_*.srt` 合并成最终 `srt` 时使用。

若本次流程没有按块拆分，就不要调用这个脚本，也不要为了适配脚本而强行拆分字幕文件。

常用方式：

```bash
python3 scripts/merge_clean_srt.py raw.srt chunks_dir clean.srt
python3 scripts/merge_clean_srt.py raw.srt chunks_dir clean.srt --allowed-deletions '284,415,450'
```

## 交付要求

交付时至少说明：

- 输出文件路径
- 是否保留原始文件未覆盖
- 是否存在允许删除条目
- 是否执行了校验脚本
- 是否仍有需要人工回听复核的告警位点
