# 原始要求

我是一个专业的纪录片视频后期，对于视频字幕，我已经基于音频识别工具进行了asr的识别，现在需要对其进行加工，成为符合上线要求的严谨的字幕文件，要求如下：
1. 修正识别错误，包括正确的"的地得"，合理的"他她它"；
2. 一些 嗯 啊 呃 哈之类的语气词删除，不能有标点符号；
3. 子句间需要有必要的空格停顿：
    举例1: "来那你就先开始"，合理的拆分为 "来 那你就先开始"
    举例2: "你是第二个是吧"，合理的拆分为 "你是第二个 是吧"
4. 删除重复字眼，不要书面化改写，不要总结，不要扩写，只做字幕级清洗和纠错;
5. 每一条字幕，最长不超过14个字，单条字幕内不要换行;
6. 如果原字幕超过14个字，注意准确的断句拆分，
    举例1: "还得谢谢各位母亲对家里的付出"，较好的拆分为“还得谢谢各位母亲” + “对家里的付出”
7. 不得因文本纠错、删减语气词、删除重复字眼或拆分字幕等，造成后续字幕整体提前或滞后。所有调整应限于当前字幕条目及其局部范围内。

## 关于时间轴处理要求：

1. 处理后字幕必须保持与原音频严格同步，不得造成后续字幕整体前移或后移，不得出现全局时间漂移。
2. 若仅进行字幕级清洗与纠错（包括识别错误修正、删除语气词、删除明显重复字眼、补空格停顿等），且未改变原句语义边界，则默认保持该条字幕原始开始时间与结束时间不变。
3. 若因断句或单条超过 14 字而拆分字幕，则拆分后的所有字幕时间区间必须完全落在原字幕原始时间范围内；各子条目之间不得重叠，优先首尾衔接；总覆盖时长必须与原字幕原始时长一致。
4. 拆分后的时间分配应优先依据语义停顿和说话节奏，无法精确判断时，再按各子句有效字符数比例分配时长，不得机械平均切分。
5. 删除“嗯、啊、呃、哈”等无意义语气词或删除明显重复字眼时，若其位于句中，通常不调整原字幕起止时间；若该内容本身构成独立且无意义的字幕条目，则可直接删除该条，但不得将前后字幕无依据地拉伸覆盖。
6. 每条字幕应尽量贴合对应语音片段，字幕不得明显早于语音出现，也不得在语音结束后明显滞留；如允许微调，单条字幕起止时间相对语音边界的偏差应尽量控制在 ±100 至 150 毫秒以内。
7. 拆分和删改后，仍须保证字幕可读性，避免出现过短闪现、明显漏挂、无意义空窗或破坏自然语义边界的切分。

## 一些额外的 checklist：
1. 不要在处理时，误删除有意义的字幕条目；
2. 字幕的时间轴 double check；

如果字幕文件过长，尽可能以并行多任务拆分 subagents 的方式调度执行。不要覆盖原始的字幕文件，方便做校准。
